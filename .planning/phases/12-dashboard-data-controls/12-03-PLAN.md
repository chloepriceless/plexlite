2. **`groupScheduleRulesForDashboard(rules)`** -- Port from app.js lines 1083-1116. Takes flat array of rules (each has target, start, end, value), groups by `start|end` timeslot key into objects with `{ start, end, grid, charge, stopSocPct, enabled, ruleId, source, displayTone, autoManaged, activeDate }`. The `grid` field comes from `target === 'gridSetpointW'`, `charge` from `target === 'chargeCurrentA'`. Returns array of grouped slots. Handles null/non-array input by returning [].

3. **`collectScheduleRulesFromRowState(rows)`** -- Port from app.js lines 1020-1081. Takes array of row objects (from UI edit state), produces flat array of API-compatible rules. For each row:
   - Skip if no start/end
   - If gridEnabled and gridVal is finite: create `{ id: 'grid_N', enabled, target: 'gridSetpointW', start, end, value: gridVal }`. If stopSocPct is finite, add `stopSocPct`.
   - If chargeEnabled and chargeVal is finite: create `{ id: 'charge_N', enabled, target: 'chargeCurrentA', start, end, value: chargeVal }`.
   - Copy source, autoManaged, displayTone, activeDate if present.

All functions use `export function`.
