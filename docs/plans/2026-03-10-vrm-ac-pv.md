# VRM AC PV and PV Inverter Import Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Den VRM-Import und die History so erweitern, dass AC-gekoppelte PV-Anlagen und PV-Inverter-Rohdaten aus VRM gespeichert, in kanonische PV-Summen uebernommen und in der UI sichtbar werden.

**Architecture:** Der bestehende VRM-Stats-Import bleibt die Grundlage, wird aber um AC-PV-Codes und optional verfuegbare PV-Inverter-Rohdaten erweitert. Kanonische PV-Summen werden additiv aus DC- und AC-PV aufgebaut, waehrend Rohdaten getrennt als Debug-/Vergleichsbasis erhalten bleiben. Runtime und UI nutzen die neuen Serien, ohne das bestehende Fallback-Modell fuer Anlagen ohne AC-PV zu brechen.

**Tech Stack:** Node.js, `node:test`, Vanilla JS, SQLite
